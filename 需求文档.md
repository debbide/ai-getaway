# AI Gateway 中转系统需求文档

## 一、项目目标

开发一个轻量级 AI API 中转系统。

系统需要兼容 OpenAI API 格式。

系统核心逻辑：

- 用户使用平台生成的 API Key 调用接口
- 系统根据用户绑定关系
- 自动转发到对应的上游账号 API Key
- 实现"一用户一上游账号"
- 用户之间完全隔离

本系统不是传统按量计费中转。

本系统属于：

- 多账号路由
- 多租户 AI 网关
- 订阅制账号池中转

## 三、用户业务流程

### 3.1 整体流程

```
用户注册 → 选择套餐 → 完成付费 → 管理员审核(去上游开账户) → 审核通过(配置上游渠道和API Key) → 用户创建API Key → 获得订阅服务
```
用户注册 → 选择套餐 → 完成付费 → 管理员审核(去上游开账户) → 审核通过（配置对应的上游渠道和apikey） → 用户创建自己的API Key → 获得订阅服务
```

### 3.2 详细步骤

#### 第一步：用户注册

- 用户访问首页
- 点击"注册"按钮，弹出模态框
- 填写：用户名、邮箱、密码
- 提交后进入"待审核"状态

#### 第二步：选择套餐

- 登录后，用户进入套餐选择页面
- 展示可用套餐列表（如：基础版、专业版、企业版）
- 每个套餐显示：价格、额度、有效期

#### 第三步：完成付费

- 用户选择套餐后，进入支付页面
- 支持在线支付方式（预留接口）
- 支付完成后，订单进入"待审核"状态

#### 第四步：管理员审核

- 管理员登录管理后台
- 查看用户的支付订单
- 管理员到上游服务商开通对应的账户
- 将上游账户信息（渠道+API Key）绑定到本系统
- 点击"审核通过"

#### 第五步：审核通过

- 管理员配置该用户的对应上游渠道和 API Key
- 用户收到审核通过通知
- 用户可以开始使用 AI API 服务

## 四、功能模块

### 4.1 用户端

| 模块 | 功能 |
|------|------|
| 首页 | 精美展示页面，含产品介绍、定价、特性等 |
| 登录 | 模态弹窗，邮箱/密码登录 |
| 注册 | 模态弹窗，用户名/邮箱/密码注册 |
| 套餐选择 | 展示套餐列表，选择适合的方案 |
| 支付 | 生成订单，接入支付接口 |
| API Key 管理 | 创建、查看、禁用 API Key（关联管理员配置的上游渠道） |
| 用量统计 | 查看已用额度、剩余额度 |
| 个人中心 | 修改密码、查看订单记录 |

### 4.2 管理端

| 模块 | 功能 |
|------|------|
| 用户管理 | 查看用户列表、用户详情 |
| 订单管理 | 查看订单、审核操作 |
| 上游账户 | 添加/管理上游账户 |
| 套餐管理 | 添加/修改套餐 |
| API Key 管理 | 查看所有用户的 API Key |
| 统计分析 | 用户量、订单量、调用量统计 |

## 五、前端UI要求

### 5.1 首页

- 精美的落地页设计
- Hero 区域：大标题 + CTA 按钮
- 功能特性展示区
- 定价套餐展示区
- 页脚含联系方式

### 5.2 登录/注册

- 所有登录/注册操作使用**模态弹窗**
- 不跳转到独立页面
- 弹窗内切换登录/注册表单
- 表单验证和错误提示

### 5.3 设计风格

- 现代简约风格
- 暗色主题为主
- 渐变色点缀
- 圆角卡片设计
